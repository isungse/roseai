# i18n (KO / EN)

`next-intl` 기반. 한국어(`ko`)가 기본, 영어(`en`)는 병렬.

## 구조

```
messages/
  ko.json
  en.json
app/
  [locale]/
    layout.tsx          # NextIntlClientProvider
    page.tsx
i18n/
  routing.ts            # defineRouting({ locales: ['ko','en'], defaultLocale: 'ko' })
  request.ts            # getRequestConfig
middleware.ts           # createMiddleware(routing)
```

## 번역 키 규칙

- 네임스페이스는 **섹션 또는 컴포넌트 단위**. ROSE-AI 홈페이지 기준: `hero`, `modules`, `philosophy`, `evidence`, `contact`, `common`.
- 키는 dot.camelCase. 공백·하이픈·대문자 금지.
- 같은 문구라도 맥락이 다르면 키를 분리한다. "Submit" 버튼과 "Submit" 폼 타이틀은 다른 키.
- 플레이스홀더는 ICU MessageFormat: `"hello": "안녕하세요 {name}님"`.
- 복수형은 ICU plural: `"items": "{count, plural, =0 {항목 없음} one {1개 항목} other {#개 항목}}"`.

## 데이터 상수 vs 번역 텍스트 분리

번역 가능한 텍스트는 `messages/*.json` 에, 번역 불변 메타는 `lib/data/*.ts` 에 둔다.

예 — 항목별 데이터/번역 분리 패턴:

```ts
// lib/data/modules.ts
export const MODULES = [
  { id: "01", i18nKey: "<moduleKey>", stack: ["<TECH 1>", "<TECH 2>", ...] },
  // …
] as const;
```

```json
// messages/ko.json
{ "modules": { "<moduleKey>": { "title": "…", "em": "…", "desc": "…" } } }
```

`<moduleKey>` 는 모듈 정체성을 반영한 camelCase 식별자 (예: `serviceDesk`, `mobileSolution`). 모듈 정체성이 바뀌면 키 이름도 같이 리네이밍해 슬롯명이 콘텐츠와 어긋나지 않게 유지.

원칙:
- `id`, `stack`, 이미지 경로, 숫자값은 데이터 상수. 로케일 불문 동일.
- `title`, `subtitle`, `desc` 등 사람이 읽는 문자열은 JSON.
- 디자인 요소인 언어 혼합 표기 (Contact 의 `"01 / COMPANY · 회사"`) 는 **디자인이지 번역 대상이 아니다.** 로케일 불문 그대로.

## 단일 로케일 단일 페이지

- `/ko` 페이지는 한국어만, `/en` 페이지는 영어만 렌더한다. 한 페이지에 두 언어를 동시에 표시하는 bilingual 모드는 **사용하지 않는다**.
- 언어 전환 = URL 전환. `LangToggle` 은 `usePathname()` + `useRouter().replace(pathname, { locale: 'en' })` 로 구현.
- Hero sub-headline 처럼 원본이 ko/en 을 동시 표기하던 부분은 각 로케일 JSON에 해당 언어 한 줄만 넣는다.

## 하드코딩 금지

- JSX 안의 사용자 노출 텍스트는 전부 `useTranslations()` 또는 `getTranslations()` 를 통과해야 한다.
- 로그/콘솔/에러 메시지(개발자용)는 예외 — 하드코딩 허용.
- `aria-label`, `title`, `alt` 도 번역 대상이다.

## 두 JSON 파일 동기화

- `ko.json` 과 `en.json` 의 **키 트리는 항상 동일**해야 한다.
- 추가/삭제/이름변경은 두 파일에 동시에 반영. 일시적으로 한 쪽만 비워두지 않는다.
- 누락 키 감지용 스크립트는 향후 추가 (확장 포인트).

### 의도된 빈 문자열 값

한 쪽 로케일에만 존재해야 하는 **디자인 요소**(예: KO 에만 표시하는 한글 부제 `contact.headingEm`)는 반대쪽 로케일에서 키를 빼지 말고 **`""` 로 명시**한다. 렌더 쪽 컴포넌트는 `const em = t("headingEm"); em && <em>{em}</em>` 패턴으로 처리하고, **이유를 한 줄 주석으로 남긴다** — 나중에 "빈 값이니 지워도 되겠지" 오해를 방지 (`components/sections/ContactSection.tsx` 참조).

### 멀티-라인 본문 값

`desc` 등 사람이 읽는 본문이 여러 줄로 분리되어 보여야 하는 경우, JSON 값에 `\n` 을 넣고 렌더 컴포넌트에 `whitespace-pre-line` 클래스를 적용한다.

```json
"desc": "첫 번째 문장.\n두 번째 문장."
```

```tsx
<p className="whitespace-pre-line">{t(`${key}.desc`)}</p>
```

`whitespace-pre-line` 은 `\n` 만 줄바꿈으로 렌더하고 들여쓰기 공백은 collapse — 단락 구조만 살린다. JSON 측은 문장당 한 줄로 쓸 수 있어 가독성·diff 편의 양쪽에서 이득. 단락별로 시각·의미적으로 완전히 분리해야 한다면 desc 를 배열로 모델링하고 `<p>` 를 여러 개 렌더하는 쪽이 더 명확.

## 라우팅

- URL: `/ko/...`, `/en/...`. 루트 `/` 는 미들웨어가 `/ko` 로 리다이렉트.
- 언어 전환은 경로만 바꾸고 쿼리·해시는 보존.
- `<Link>` / `useRouter` 는 `next-intl` 이 제공하는 래퍼를 사용. 네이티브 `next/link` 직접 사용 금지 (locale prefix 누락 원인).

### 미들웨어 매처와 `/public` 정적 자산

`middleware.ts` 의 매처 `/((?!api|_next|_vercel|.*\..*).*)` 는 **점이 없는 경로를 전부** 잡는다. `/demos/service-desk/` 같은 디렉토리형 URL 은 미들웨어가 가로채 `/ko/` 프리픽스를 붙이고, 매칭되는 `[locale]` 라우트가 없으면 동적 컴파일을 80s+ 시도하다 404 — 브라우저는 무한 로딩으로 보인다. 추가로 Next 16 의 정적 핸들러는 **디렉토리 인덱스를 자동 해석하지 않는다** (`/foo/` → `/foo/index.html` 자동 매핑 없음).

규칙:

- `/public/` 정적 자산을 외부에 노출할 때는 **URL 에 확장자를 명시**한다 (`/demos/service-desk/index.html`, `/og/ko.png`). 점이 들어가면 매처의 `.*\..*` 가 자동 제외 → Next 정적 핸들러가 직접 서빙.
- 디렉토리 형태 URL 이 꼭 필요하면 매처 화이트리스트에 추가하거나 `next.config.ts` 의 `rewrites()` 로 매핑.
- 브라우저는 308 Permanent Redirect 를 강하게 캐시한다. 잘못된 trailing-slash 응답이 한 번 캐시되면 URL 을 고쳐도 사용자 측에선 hang 으로 재현될 수 있음 — 디버깅 시 hard reload (`Ctrl+Shift+R`) 우선.

(`components/sections/ModuleTabs.tsx` + `lib/data/modules.ts` 의 `demoHref` 패턴 참조.)

## 메타데이터

- `generateMetadata` 안에서 `getTranslations({ locale, namespace })` 로 제목/설명을 가져온다.
- OG 이미지 경로도 로케일별로 분기 (`/og/ko.png`, `/og/en.png`).
- `alternates.languages` 에 양쪽 경로 명시 (SEO 참조).

## 날짜·숫자·통화

- `next-intl` 의 `useFormatter()` 사용. `toLocaleString` 직접 호출 금지.
- 통화 기본: KO → KRW, EN → USD.
- 시간대는 Asia/Seoul 기준, UTC 저장·표시 시 변환.
