# i18n (KO / EN)

`next-intl` 기반. 한국어(`ko`)가 기본, 영어(`en`)는 병렬.

## 구조

```
messages/
  ko.json
  en.json
app/
  [locale]/
    layout.tsx          # NextIntlClientProvider
    page.tsx
i18n/
  routing.ts            # defineRouting({ locales: ['ko','en'], defaultLocale: 'ko' })
  request.ts            # getRequestConfig
middleware.ts           # createMiddleware(routing)
```

## 번역 키 규칙

- 네임스페이스는 **섹션 또는 컴포넌트 단위**. ROSE-AI 홈페이지 기준: `hero`, `modules`, `philosophy`, `evidence`, `contact`, `common`.
- 키는 dot.camelCase. 공백·하이픈·대문자 금지.
- 같은 문구라도 맥락이 다르면 키를 분리한다. "Submit" 버튼과 "Submit" 폼 타이틀은 다른 키.
- 플레이스홀더는 ICU MessageFormat: `"hello": "안녕하세요 {name}님"`.
- 복수형은 ICU plural: `"items": "{count, plural, =0 {항목 없음} one {1개 항목} other {#개 항목}}"`.

## 데이터 상수 vs 번역 텍스트 분리

번역 가능한 텍스트는 `messages/*.json` 에, 번역 불변 메타는 `lib/data/*.ts` 에 둔다.

예 (모듈 5개):

```ts
// lib/data/modules.ts
export const MODULES = [
  { id: "01", i18nKey: "workforcePlanner", stack: ["NEXT.JS 15","POSTGRES","REDIS","PYTHON · PROPHET","WEBSOCKET"] },
  // …
] as const;
```

```json
// messages/ko.json
{ "modules": { "workforcePlanner": { "title": "Workforce Planner", "subtitle": "인력 운영 관제 — People Control Plane", "desc": "…" } } }
```

원칙:
- `id`, `stack`, 이미지 경로, 숫자값은 데이터 상수. 로케일 불문 동일.
- `title`, `subtitle`, `desc` 등 사람이 읽는 문자열은 JSON.
- 디자인 요소인 언어 혼합 표기 (Contact 의 `"01 / COMPANY · 회사"`) 는 **디자인이지 번역 대상이 아니다.** 로케일 불문 그대로.

## 단일 로케일 단일 페이지

- `/ko` 페이지는 한국어만, `/en` 페이지는 영어만 렌더한다. 한 페이지에 두 언어를 동시에 표시하는 bilingual 모드는 **사용하지 않는다**.
- 언어 전환 = URL 전환. `LangToggle` 은 `usePathname()` + `useRouter().replace(pathname, { locale: 'en' })` 로 구현.
- Hero sub-headline 처럼 원본이 ko/en 을 동시 표기하던 부분은 각 로케일 JSON에 해당 언어 한 줄만 넣는다.

## 하드코딩 금지

- JSX 안의 사용자 노출 텍스트는 전부 `useTranslations()` 또는 `getTranslations()` 를 통과해야 한다.
- 로그/콘솔/에러 메시지(개발자용)는 예외 — 하드코딩 허용.
- `aria-label`, `title`, `alt` 도 번역 대상이다.

## 두 JSON 파일 동기화

- `ko.json` 과 `en.json` 의 **키 트리는 항상 동일**해야 한다.
- 추가/삭제/이름변경은 두 파일에 동시에 반영. 일시적으로 한 쪽만 비워두지 않는다.
- 누락 키 감지용 스크립트는 향후 추가 (확장 포인트).

## 라우팅

- URL: `/ko/...`, `/en/...`. 루트 `/` 는 미들웨어가 `/ko` 로 리다이렉트.
- 언어 전환은 경로만 바꾸고 쿼리·해시는 보존.
- `<Link>` / `useRouter` 는 `next-intl` 이 제공하는 래퍼를 사용. 네이티브 `next/link` 직접 사용 금지 (locale prefix 누락 원인).

## 메타데이터

- `generateMetadata` 안에서 `getTranslations({ locale, namespace })` 로 제목/설명을 가져온다.
- OG 이미지 경로도 로케일별로 분기 (`/og/ko.png`, `/og/en.png`).
- `alternates.languages` 에 양쪽 경로 명시 (SEO 참조).

## 날짜·숫자·통화

- `next-intl` 의 `useFormatter()` 사용. `toLocaleString` 직접 호출 금지.
- 통화 기본: KO → KRW, EN → USD.
- 시간대는 Asia/Seoul 기준, UTC 저장·표시 시 변환.
